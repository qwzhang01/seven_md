## Context

Seven Markdown 项目处于活跃开发阶段，需要一个集中的文档来跟踪未来计划开发的功能和技术改进方向，便于团队规划和社区贡献者参与。

## Goals / Non-Goals

**Goals:**
- 创建 `docs/FUTURE_TODO.md` 文档
- 记录已规划的功能
- 记录技术债务和优化方向
- 便于团队长期规划

**Non-Goals:**
- 不是开发计划（不需要时间线）
- 不是需求文档（不需要详细规格）
- 不是 changelog（不记录已完成的功能）

## Document Structure

```markdown
# Seven Markdown 未来规划 / TODO

## 一、已规划的功能
### v2.x 版本计划
- [ ] Git 集成
- [ ] 多窗口支持
...

## 二、技术改进
### 性能优化
- [ ] 编辑器虚拟滚动
...

## 三、技术债务
- [ ] 单元测试覆盖率提升
...

## 四、第三方集成
- [ ] Obsidian vault 导入
...

## 五、平台支持
- [ ] Windows 版本
...
```

## Update Policy

- 新功能规划在此讨论和记录
- 实现完成后移动到 CHANGELOG.md
- 定期（如每月）回顾和更新
