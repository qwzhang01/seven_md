## Why

当前资源管理器中，点击"新建文件"按钮始终在根目录创建文件，忽略了用户当前选中的文件夹上下文；同时文件树缺乏拖拽移动能力，用户无法通过直觉性的拖拽操作来整理文件和文件夹结构。这两个问题导致文件管理体验与主流编辑器（VS Code、Obsidian 等）存在明显差距。

## What Changes

- **文件夹感知新建**：点击某个文件夹（选中状态）后，点击"新建文件"或"新建文件夹"按钮，新条目应创建在该文件夹内，而非根目录
- **文件拖拽移动**：文件可拖拽到任意文件夹中，实现跨文件夹移动
- **文件夹拖拽移动**：文件夹可拖拽到其他文件夹内，实现嵌套层级调整（不允许拖入自身或其子孙文件夹）
- **拖拽视觉反馈**：拖拽过程中高亮目标文件夹，提供清晰的放置目标指示
- **拖拽到根目录**：支持将文件/文件夹拖拽到文件树顶部空白区域，移动到根目录

## Capabilities

### New Capabilities

- `folder-context-create`: 文件夹感知的新建文件/文件夹——在选中文件夹的上下文中创建新条目
- `explorer-drag-and-drop`: 资源管理器文件树的拖拽移动能力——支持文件和文件夹的拖拽重新排列

### Modified Capabilities

- `sidebar-explorer`: 新建文件/文件夹的触发逻辑需感知当前选中文件夹，行为需求发生变化
- `file-operations`: 文件移动（move）操作作为新的文件操作类型需要被支持

## Impact

- **前端组件**：`FileTree` / `ExplorerPanel` 组件需要增加拖拽事件处理和选中文件夹状态跟踪
- **状态管理**：`workspace-store` 需要支持文件/文件夹移动操作（更新路径、重命名引用）
- **文件系统**：需要调用 Tauri 的文件系统 API 执行实际的 `rename`（移动）操作
- **Tab 管理**：移动文件后，已打开的 Tab 需要同步更新文件路径引用
- **无破坏性变更**：现有新建文件行为在无选中文件夹时保持不变（回退到根目录）
